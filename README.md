# Situação de Aprendizagem - Projeto Train.Info

### Contexto do projeto:

Durante o ano de 2025, o terceiro ano do ensino médio da Escola SESI de Referência foi desafiada a montar e implementar um site de monitoramento e suporte para uma empresa de trens. Assim surgiu o Train.Info, um site projetado e desenvolvido pelos seguintes desenvolvedores:

O presente projeto se desenvolveu em um site com sistema de login para funcionários e controle dos mesmos, onde é possível não apenas o monitoramento dos trens, mas também a garantia de segurança dos veículos e dos maquinistas que os operam.

Foram utilizadas diversas tecnologias, como um sistema de banco de dados SQL para controle das informações dos funcionários, trens, manutenções, rotas e relatórios gráficos do desempenho da empresa.

O site foi programado em PHP e JavaScript, para que pudesse atender a todas as demandas da empresa, tendo um desing responsivo, organizado e intuitivo.

Devido à necessidade de diversas funcionalidades, como monitoramento em tempo real dos trens, relatórios dos maquinistas sobre problemas e notificações em caso de crises, o site conta atualmente com conexão em tempo real por IoT, garantindo que tudo esteja amplamente conectado.

Com todas essas necessidades, tivemos um repositório vasto e bem estruturado, onde cada desenvolvedor ficou responsável por uma parte específica do site, separando as tarefas e reunindo tudo para a criação do sistema como um todo. Os documentos, por questão de boas práticas, estão separados por tipo e nomeados com suas respectivas funções.

---

### Objetivo do projeto:

> O objetivo principal do projeto Train.Info foi desenvolver e implementar um site abrangente de monitoramento e suporte para uma empresa de trens, como parte de um desafio proposto à turma de Desenvolvimento de Sistemas 24M6 da Escola SESI de Referência, durante o ano de 2025.<br>
<br>

<b> *Os propósitos específicos do projeto incluíram:* </b>

<b>Praticar o monitoramento e segurança:</b> Criar uma plataforma que permitisse não apenas o monitoramento dos trens em tempo real, mas também garantisse a segurança dos veículos e dos maquinistas.<br>

<b>Desenvolver uma gestão de funcionários e Dados:</b> Implementamos um sistema de login e controle de funcionários, utilizando um banco de dados SQL para gerenciar informações sobre colaboradores, trens, manutenções, rotas e relatórios de desempenho.<br>

<b>Usar conectividade em tempo real:</b> Utilizamos tecnologia de Internet das Coisas (IoT) para permitir o monitoramento em tempo real dos trens e a mecanismos dos mesmos.<br>

<b>Criar funcionalidades e usabilidade:</b> Desenvolvemos o site utilizando PHP e JavaScript para garantir que fosse visualmente agradável, responsivo e funcional.<br>

Em resumo, o projeto visou criar uma solução completa para avaliar os conhecimentos da equipe diante o desafio.

---

### Funcionalidades Principais:

Gerenciamento Completo de Dados (CRUD): Permite criar, visualizar, atualizar e excluir informações essenciais como trens, rotas, usuários, viagens, sensores, notificações e mais.

Adequação a Regras de Negócio: Todas as informações seguem padrões de validação e formatação previamente documentados, garantindo consistência e organização.

Interface Web Responsiva: Acessível diretamente pelo navegador, permitindo que gestores e usuários interajam com todas as funcionalidades sem necessidade de instalação adicional.

Integração com Hardware: Suporte para conexão com módulos ESP32, permitindo integração com sensores, atuadores e outros dispositivos IoT para monitoramento e controle.

### Informações Complementares:

> Para configurar e instalar o projeto, siga esse passo a passo:

##### 1. Clonar o repositório:

Em seu "Prompt de Comando", execute os seguintes comandos:

> git clone https://github.com/helenacani2/SA-mobilidade-urbana_DS.git<br>
> cd SA-mobilidade-urbana_DS

##### 2. Configurar servidor:

Nessa etapa, se move o código para o local onde o servidor Apache pode executá-lo.

> a) No seu software de servidor local (XAMPP), localize a pasta raiz do servidor<br>
> (No XAMPP o diretório padrão é geralmente C:\xampp\htdocs\)<br>
> b) Mova a pasta SA-mobilidade-urbana_DS (que você clonou no Passo 1) para dentro do htdocs<br>
> c) Inicie os serviços Apache e MySQL através do painel de controle do XAMPP<br>

##### 3. Configuração do Banco de Dados (MySQL)
> Abra o seu navegador e acesse a interface de gerenciamento do banco de dados em http://localhost/phpmyadmin<br>
> Crie o banco de dados usando os arquivos da página -sql, selecione e crie a população do banco, também incluída na pasta -sql<br>

##### 4. Ajustar a Conexão PHP
> Localize o arquivo de conexão do banco de dados do seu projeto "train_info_bd.php", dentro da pasta -php<br>
> Edite este arquivo para que as credenciais de conexão correspondam às suas configurações locais:<br>

<br>

> $host = "localhost";<br>
> $usuario = "TROCAR";<br>
> $senha = "TROCAR";<br>
> $banco = "NOME_QUE_VOCÊ_ESCOLHEU";<br>

<br>

Com isso, será possível implementar e instalar o sistema corretamente!
